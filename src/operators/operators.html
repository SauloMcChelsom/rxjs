operators

empty()
  • Este operador estático é útil para criar um Observable simples que só emite a notificação completa.
  • Por exemplo, operadores, of(arg), from(arg)... precisão passar argumentos para criar um Observable, diferente do empty(). Tambem ultilizado para criar um retorno vazio
startWith()
  • Primeiro emita os valores do startWith(S), depois emita os valores do observable de origem.
  • valores do startWith() => S
  • valores do observable de origem => A, B, C
from()
  • Cria um Observable a partir de um Array
  • Converte quase tudo em um Observable
  • Converte um array em um Observable
range()
  • operador de intervalo emite um intervalo de inteiros sequenciais, em ordem, onde você seleciona o início do intervalo e seu comprimento. Por padrão, não usa SchedulerLike e apenas entrega as notificações de forma síncrona, mas pode usar um SchedulerLike opcional para regular essas entregas.
timer()
  • Emite números crescentes, um a cada segundo (1000ms), iniciando após 3 segundos
interval()
  • Emite números crescentes, um a cada segundo (1000ms) até o número 3
concat()
  • Concatena vários Observables juntos, emitindo sequencialmente seus valores, um Observable após o outro.
  • Cria um Observable de saída que emite sequencialmente todos os valores de um determinado Observable e depois passa para o próximo.
  • A -> B ->|
    X -> Y ->|
    concat
    A -> B -> X -> Y ->
forkJoin()
  • Aguarde a conclusão dos Observables e, em seguida, combine os últimos valores que eles emitiram.
merge()
  • Achata vários Observables juntos, misturando seus valores em um Observable.
  • direrente do concat() que espera a sequencia do termino de um, para começar o outro, aqui no merge() tudo junto e misturado
partition()
  • partition outputs an array with two Observables that partition the values from the source Observable through the given predicate function..
  • Dividi em dois Observables por meio de uma condição
